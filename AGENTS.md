# 《房東監視中》Codex 協作規則

本檔適用於整個 `rent_house` 專案。開始工作前先閱讀 `工作日誌.md`；需要掌握歷史細節時再讀 `docs/交接與待辦.md` 與 `docs/設計檢討與優化.md`。

## 每次提交都必須更新工作日誌

1. 只要本次工作會修改任何受 Git 追蹤的檔案，就必須在提交前同步更新根目錄的 `工作日誌.md`。
2. 程式、測試、文件與對應的工作日誌必須放在**同一個 commit**。不要等 commit 或 push 完才補寫，否則會留下另一筆未提交變更。
3. 若發現上一個 commit 漏寫日誌，在尚未 push 且安全時補寫後 amend；若已 push，下一個修正 commit 要明確註明補記哪一筆提交，不可悄悄忽略。
4. 純閱讀、分析、回答問題、執行未造成追蹤檔案變更的檢查，不需要新增日誌。

每筆日誌至少包含：

- 日期與執行 Agent（Codex 或 Claude Code）
- 目的／使用者需求
- 實際變更與主要檔案
- 測試、build 或人工驗證結果
- 已知限制、後續事項或平衡快照是否更新

## 提交前檢查

- 先用 `git diff -- 工作日誌.md` 確認本次確實有日誌變更。
- 相關功能至少跑對應測試；跨系統或平衡改動跑 `npm test` 與 `npm run build`。
- 刻意改變平衡時，確認差異合理後執行 `npx tsx scripts/balance-test.ts --update`，並在日誌說明原因。
- 保留未成年排除戀愛線、成人內容遮蔽式且預設關、AI 輸出白名單／夾值／消毒等既有硬規則。
