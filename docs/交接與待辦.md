## 二、不可動搖的原則(先讀,違反 = 直接回絕使用者)

1. **未成年排除戀愛線**:`canRomance()` 把關;特邀角色(如大雄/靜香)以**原作年齡**判定,
   `isAdult !== true` 一律拒絕,**不接受「當作成年」的指示**(使用者曾要求過,已回絕,立場不變)。
2. **AI 永不生成露骨內容**:🔞 互動全部走本地模板 + 遮蔽式演出(sprite 隱藏 + 霧氣/關燈 fx),
   worker 的事件 schema 明文告知 AI「親密內容不在清單內,不要嘗試」。adultMode 只解鎖
   「成年情侶」的含蓄互動,預設關。
3. **AI 輸出一律消毒**:sanitizeAiEvent / sanitizeDirective / sanitizeArcUpdate / 白名單 interaction id,
   AI 不能發明機制、數值一律 delta 且夾值。

## 三、開發流程(照抄可用)

```powershell
# PowerShell 每個 session 要先補 node 路徑
$env:Path = "C:\Program Files\nodejs;$env:Path"
cd C:\Users\User\claude_try\rent_house

npx tsx scripts/<某>-test.ts      # 單支測試(無頭,scripts/ 下 50+ 支)
npx tsx scripts/sim-trace.ts      # 24h 模擬追蹤,必須 0 異常
npx tsx scripts/balance-test.ts   # 固定種子 10 天快照;刻意的平衡改動 → 加 --update 重建基準
npm run build                     # vite build(也是型別檢查)
```

- **無頭測試模板**:檔頭先 mock `localStorage`(`(globalThis as any).localStorage = {...}`),
  再 `await import("../src/store")`。mock AI 用 `setNarrateImplForTest()`、`diaryTiming.gapMs = 1`。
- **commit**:訊息繁中、結尾 `Co-Authored-By: Claude <模型名> <noreply@anthropic.com>`。
- **部署驗證**:Bash 的 curl 在此機沙盒無網路;用 PowerShell:
  ```powershell
  $r = Invoke-WebRequest -Uri "https://rent-house.guai7485962.workers.dev" -UseBasicParsing
  # 取 assets/index-*.js 檔名,再抓該 js;中文字串要用 UTF8 解碼才搜得到:
  $b = (Invoke-WebRequest -Uri "<js url>" -UseBasicParsing).RawContentStream.ToArray()
  [System.Text.Encoding]::UTF8.GetString($b).Contains("某新字串")
  ```
- **改存檔結構**:`SAVE_VERSION++` + 在 `persistence.ts` 的 `MIGRATIONS` 加一段升級函式;
  新欄位用 `?? 預設值` 讀取通常就夠(本批 pets/diaryHour 都這樣做,沒升版)。

## 八、第三批待辦(2026-07-12 最終 review:API 濫用防護 + 勘誤)

### I. worker 端點防護(建議提前到 E-1 之後做——影響的是「大家的額度」)

`/api/narrate`、`/api/invite` 是**公開端點**:無來源檢查、無速率限制、無請求大小上限。
任何人拿到網址就能直接 POST 刷掉共用的 Gemini 免費額度(RPD 用盡 → 所有玩家整天全模板);
若 Cloudflare 配了 ANTHROPIC_API_KEY,備援路徑還會產生真金白銀的費用。分層防護(由便宜到貴):

1. 🟢 **同源檢查**:worker 檢查 `Origin`/`Referer` 是否為本站,不符直接 403。
   擋不了偽造 header 的攻擊者,但擋掉 99% 的順手濫用,一行 if 的成本。
2. 🟢 **請求體上限 + 欄位夾值**:body 超過 ~16KB 直接拒;NarrateCtx 的陣列欄位
   (todayLog/relationships/neighbors/flags)server 端 slice 上限、字串欄位截長——
   現在是前端組多長就餵多長,惡意 payload 可以灌爆 prompt(= 灌爆 token 成本)。
   同場加映:`parseResult` 的 `diary` 沒截長,回填前 slice(如 500 字)。
3. 🟡 **速率限制**:Cloudflare 免費方案有 Rate Limiting 規則(dashboard 設,零程式碼),
   對 /api/* 設每 IP 每分鐘上限(如 10 req);程式內做法是 Workers KV/Durable Object
   計數器,成本較高,先用 dashboard 規則就夠。
4. 🔴 **玩家自帶 key**(= 第一批 C-10):根治共用額度問題,順便把費用歸戶。
