# 《房東監視中》開發工作日誌

> 更新日期:2026-07-10
> 一款「異步掛機觀察型」直式手機模擬遊戲:玩家是房東,透過俯視娃娃屋觀察租客生活、佈置家具、招租收租;劇情由 **AI(Gemini)依歷史即時生成**,並能影響走向。

## 線上位置
- **遊戲網址(手機可玩):** https://rent-house.guai7485962.workers.dev
- **Repo:** github.com/guai7485962/rent-house(push 到 `main` → Cloudflare 自動 build + deploy)
- **架構:** 靜態前端(Vue+Vite)+ Cloudflare Worker(同時服務網站與 `/api/narrate`);AI 金鑰存 Worker Secret `GEMINI_API_KEY`。

## 一、目前完成度總覽

「**觀察 + 房東經營 + AI 敘事 + 張力/社交**」的完整骨架都做完並上線了。核心體驗:掛機觀察租客 → 佈置招租收租 → 處理突發抉擇 → 看 AI 把每天寫成有記憶、會演進的故事,鄰居之間還會發展友情/戀愛/衝突。**沒金鑰/離線/超額時自動 fallback 成模板,遊戲照跑。**

## 二、已完成的系統

### A. 核心資料與美術
- `src/types.ts` — Tenant/Furniture/事件、visual_state、房間屬性、性別與戀愛取向、記憶標籤。
- 種子 `data/tenants.json`;`src/pixel/` 像素美術(俯視 3/4)、動態租客配色(依房間指派、保證不撞色)。

### B. 樓層地圖(tile 網格)
- `src/floor/map.ts` — 16×33 網格:4 套房 + 交誼廳 + 浴室(乾濕分離)+ 洗衣間 + 大門,可捲動。
- `floorScene.ts` + `FloorMap.vue` — 渲染、房間標籤、未讀/事件徽章、點擊進房/擺放/查看家具。

### C. 家具目錄 + 動態擺放
- `src/furniture/catalog.ts`(~35 件,資料驅動)、`render.ts`;`src/sim/placements.ts`(買/擺/賣/查、房間屬性、找空位)。
- 點家具 → 資訊卡 + 賣掉(退半價)`FurnitureInfo.vue`。

### D. 走動與尋路
- `pathfind.ts`(BFS + 家具障礙)、`agents.ts`(狀態驅動,走到當前活動家具旁;站立點自動避牆)。

### E. 異步掛機模擬
- `src/sim/clock.ts` — **7× 掛機時鐘(現實 1 天 = 遊戲 7 天)**,關掉再開自動補進度。
- `src/sim/routine.ts` — 24 小時作息 + 招租原型 + 家具角色反查(只用自房/共用區)。
- `src/sim/generate.ts` — 每小時觀察句(用 Fable 文本庫,隨機挑、時段安全)。
- `src/store.ts` — 每小時 tick、作息+偏離、未讀、存檔(localStorage)、快轉。

### F. 房東經營 + 收支系統
- 商店 `FurnitureShop.vue`(選→點地圖擺放)、招租 `recruit.ts`+`RecruitPanel.vue`(依裝潢契合度★)、收租。
- **收支帳** `state.ledger` + `addMoney()`(所有金錢進出單一入口)+ **每日管理費** + **財務面板 `FinancePanel.vue`(💰 收支)**:餘額/收入/支出/分類/交易明細。

### G. 張力系統(數值後果)
- **滿意度**(由心情/好感/壓力 + 房間是否滿足需求算出)。
- **突發事件抉擇**:規則式(壓力/滿意/好感觸發)+ **AI 生成**;`DecisionModal.vue` + `decide()` 套用夾值後果。
- **退租**:長期不滿 → 搬走、空房、系統通知。收租綁滿意度。

### H. 鄰居社交(完整版)
- `src/sim/social.ts` — 關係值 + 個性相容度 + 性別/取向;交誼廳相遇 → 聊天/衝突;認識→朋友→曖昧→情侶→**同居/分手**。
- **💞 關係面板**、同居抉擇彈窗、房間細看顯示感情狀態。

### I. AI 敘事(靈魂,已上線)
- `worker/index.ts` — `/api/narrate` 呼叫 **Gemini 2.5-flash**(免費層),依「當天日誌+性格+記憶+關係+抉擇」生成:
  - **每日觀察日記**(📖 卡片;AI 版加 ✨AI 標,模板版乾淨卡片)。
  - **記憶標籤**(讓歷史累積、劇情演進;去重+上限 8)。
  - **房東抉擇事件**:AI 依處境即時生成(嚴格夾值消毒),**可牽涉第二位鄰居**改動雙方數值與關係(戀情加速/室友吵架/和好…),不開放驅逐。
- `src/sim/narrate.ts` — 前端同源 fetch + 模板 fallback;供應商可切換(Gemini 優先,Claude 備援)。
- **內容變化**:`src/content/observationLines.ts`(**Fable 一次性生成**:170 句觀察 + 16 每日模板,時段安全),讓模板層與 AI 素材都不再千篇一律。

### N. 數值模型重整 + balance 快照(2026-07-10;設計檢討 §6 工作項 3、10 完成)
- **homeostasis 抗飽和**(`sim/tick.ts` applyStat):mood/stress 每小時朝「性格基準」回歸 6% 再吃活動增量,不再黏死 0/100。基準由 coreTags 決定(工作狂 stress+12、社恐+8、自律−4…),再疊三個回饋:**社交**(戀人 mood 基準+6/朋友+3/孤立−4,= socialFulfillment 簡版)、**健康**(wellbeing<30 → mood 基準−10)、**精力**(energy<25 → stress 基準+10)。affinity 是關係累積,不回歸。
- **hygiene → wellbeing(身心健康)+ 後果**:長期 stress≥80 或 energy<20 每小時蛀健康;wellbeing≤28 觸發**生病事件**(看醫生 $800 快好/送湯 $200/放著慢好);wellbeing 也有 1%/h 弱回歸避免黏 100。
- **新增 energy(精力)**:資源型——睡覺 +9/h 充電、工作/直播 −4、外出 −3…(`generate.ts` EFFECT 表);記憶漂移也接上(熬夜 −0.4、運動 +0.3)。
- **滿意度公式納入新軸**:0.25 mood + 0.3 affinity + 0.25 (100−stress) + 0.1 wellbeing + 0.1 energy,再乘房間需求滿足率。
- **存檔遷移層第一次實戰**:SAVE_VERSION 2→3,MIGRATIONS[2] 把舊檔 stats.hygiene 改名 wellbeing、補 energy;qol-test 實測 v2 舊檔匯入自動升級。
- UI:房間細看數值列加「精力/健康」兩條(共 7 條),顯示值取整;STAT_HELP 更新。
- **balance 快照測試**(工作項 10):`scripts/balance-test.ts` 固定種子(mulberry32)跑 10 遊戲日、事件自動選第一項,關鍵數值快照存 `scripts/balance-snapshot.json`;之後手感悄悄跑掉會直接 diff 現形,刻意調平衡用 `--update` 重建基準。
- 測試:新增 `stats-model-test.ts`(9 項);全部 23 個腳本 + build 通過。

### M. 調租談判(2026-07-10;設計檢討 7-1「投資遊戲性」第一項)
- 房間細看新增「💲 談房租」(僅承租人;同居者不付租談不了)→ `RentPanel.vue`:±5% 一格(-20%~+30%)、即時顯示提案金額與**模糊預感**(會感激/應會接受/有點冒險/八成翻臉)。
- 機制(`sim/tenancy.ts`):容忍度 = (滿意度×0.5+好感×0.5−35)/200(0~25%)。**降租**必成:好感/滿意↑、降 ≥10% 留 `[房東主動降租]` 記憶;**容忍內漲租**勉強接受:租金↑但好感/滿意↓;**超過容忍**拒絕:房租不變、關係惡化、不滿累積 +10、留 `[對漲租不滿]` 記憶(AI 可拿去寫劇情)。單次夾 ±30%,每人冷卻 5 遊戲日(成敗都算)。
- 新欄位 `TenantRuntime.rentChangeDay`(存檔相容:load `?? -99`);收租(月租/30)自動跟著新租金走。
- 測試:`scripts/rent-test.ts` 19 項全過。
